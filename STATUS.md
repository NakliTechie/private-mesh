## 2026-05-18 — Unit C: CRATE-PAIR endpoints on nakli-hub (Hub-only; cf-worker follows)

Per the wire-protocol audit landed in `crate-agent/docs/wire-protocol-audit.md` §5 ([`crate-agent@4748f01`](https://github.com/NakliTechie/crate-agent/commit/4748f01)), the CRATE-PAIR pairing endpoints (`/v1/pairing/intent`, `/cancel`, `/redeem`) plus the capability-lifecycle endpoints (`/v1/capability/refresh`, `DELETE /v1/capability/{id}`) per [`crate-pairing-protocol-v1.0.md`](docs/specs/crate-pairing-protocol-v1.0.md) did not exist on `nakli-hub` (or `nakli-cf-worker`). Device-pairing endpoints (`/identity/pair/*`) existed but are a separate flow. This commit closes the Hub-side gap; cf-worker parity follows in a separate milestone. **Five new endpoints** wired in [`internal/server/handlers_crate_pairing.go`](nakli-hub/internal/server/handlers_crate_pairing.go) (~370 LOC): `POST /v1/pairing/intent` (auth via macaroon; stores token payload with TTL = `expires_at - now`), `POST /v1/pairing/intent/cancel` (auth; soft-cancels unredeemed token), `POST /v1/pairing/redeem` (**unauthenticated** — the `secret` IS the auth; atomic check-and-set redemption via `UPDATE … WHERE redeemed_at IS NULL`; mints a sync-scope macaroon with caveats `time < now+1y`, `device-id == daemon_pubkey`, `operation in [read, write]`), `POST /v1/capability/refresh` (auth via current capability; re-mints preserving scope + non-time caveats with extended `time <`), `DELETE /v1/capability/{id}` (auth; piggybacks on existing `MarkGrantRevoked` so `authMiddleware`'s revocation check fires on subsequent requests). **New SQLite table** `crate_pairing_tokens` ([`nakli-hub/internal/storage/schema.go`](nakli-hub/internal/storage/schema.go) migration v2 — additive) — separate from existing `pairing_tokens` (which is for device-enrollment; different schema + semantics). Storage methods in [`nakli-hub/internal/storage/crate_pairing.go`](nakli-hub/internal/storage/crate_pairing.go) (~190 LOC) provide `CreateCratePairingToken` / `LookupCratePairingToken` / `RedeemCratePairingToken` (atomic) / `CancelCratePairingToken`. **Auth model note**: the spec's "Authorization: Identity {signed-request-from-browser}" (line 96) is interpreted as "authenticated by user's Identity-bound credential" — the existing macaroon-via-`X-Fabric-Grant` flow satisfies this; future versions could narrow with a `crate-pairing` scope caveat. **Capability shape** uses the existing `fabric-sdk-go/grant.Mint` and the existing caveat catalogue ([`fabric-spec-001-v1.0.md`](docs/specs/fabric-spec-001-v1.0.md) lines 341–358) — no new caveats needed per the audit §3. **Six new error codes** in [`envelope.go`](nakli-hub/internal/server/envelope.go): `token_format`, `token_expired` (HTTP 410), `token_not_found` (HTTP 404), `token_already_redeemed` (HTTP 409), `token_cancelled` (HTTP 404), `protocol_version` (HTTP 426). All map to E_TOKEN_* in [`crate-pairing-protocol-v1.0.md`](docs/specs/crate-pairing-protocol-v1.0.md) §"Error codes". Daemons handle unknown codes gracefully per fabric-spec line 114 — adding codes is forward-compatible. **Conformance**: [`internal/server/handlers_crate_pairing_test.go`](nakli-hub/internal/server/handlers_crate_pairing_test.go) — 13 unit tests covering the conformance matrix: intent valid / wrong version / missing secret / wrong type / expired payload / requires auth, redeem fresh / replay / unknown / expired, cancel-then-redeem, capability refresh (preserves device-id caveat), revoke-then-refresh (returns 401 `grant_revoked`). ## 2026-05-17 — M7 complete: Local Network transport (mDNS + Hub-to-Hub sync)

M7 lands the discovery half of the Local Network transport per [`local-network-spec-001-v1.1.md`](docs/specs/local-network-spec-001-v1.1.md). New package [`fabric-sdk-go/local/`](fabric-sdk-go/local/local.go) wraps `github.com/grandcat/zeroconf` with two narrow types: `Announcer` (register `_nakli-fabric._tcp.local.` with TXT records carrying hub_id / transport_id / capabilities / url) and `Browser` (long-lived browse that streams entries into an observable peer map with a 180s staleness budget; consumers can `SetExcludeTransportID(self)` to hide their own announcement). One quirk worth noting: the zeroconf v1.0.0 library closes the entries channel exactly once when its context cancels — calling `Browse` repeatedly on the same channel double-closes and panics. The Browser keeps a single long-lived call alive instead. On the Hub side, `nakli-hub serve --local-network` (with optional `--advertise-url`) announces at startup and runs a background Browser; the discovered peer set feeds `GET /fabric/v1/sync/peers` (was empty), with each entry surfacing transport_id / hub_id / url / version / capabilities / last_seen / freshness_ms. Instance names use the trailing 12 chars of the hub_id (the ULID prefix is time-encoded; two Hubs minted within the same second would otherwise collide). **Sync API became live**: `POST /fabric/v1/sync/push` accepts an `events[]` payload from a peer and applies each via the new `storage.IngestEvent` (idempotent on event_id — `ErrAlreadyPresent` is a skip not an error); `GET /fabric/v1/sync/pull?since=<cursor>&limit=N` returns events with rowid > cursor along with a next-cursor and `more` flag (via the new `storage.SyncPull`). `/sync/conflict-ack` remains 501 — full conflict surfacing lands at M7.x. **New binary** [`nakli-local-bridge/`](nakli-local-bridge/cmd/nakli-local-bridge/main.go) is a ~160-LOC standalone Go daemon: announces on mDNS (with `capabilities=discovery`), browses, and exposes `GET /local/peers` + `GET /local/health` over HTTP on `127.0.0.1:7849`. Browser tools that cannot speak mDNS query the bridge for peers. `POST /local/signal` (WebRTC relay) and `WS /local/peers/observe` (live peer streaming) ship as 501 stubs and land at M7.x. **Gate** (handoff §M7): [`./scripts/local-network-gate.sh`](scripts/local-network-gate.sh) builds nakli-hub + nakli-cli + nakli-local-bridge, starts two Hubs on free ports + the bridge, waits for mDNS to converge (up to 60s; macOS mDNSResponder can take 5-15s to propagate multi-process announcements on a single host), then drives a Hub-A-to-Hub-B sync round-trip: Hub A appends a vault event, the gate calls `/sync/pull` on Hub A and re-posts the events to `/sync/push` on Hub B, then verifies Hub B's `/sync/pull` includes the event. Bridge reports both Hubs at `GET /local/peers`. Hub A's `/sync/peers` reports Hub B. Hub-A's payload Grant doesn't verify on Hub B (each Hub holds its own root key; cross-Hub Grant trust is M7.x), so the verification reads through `/sync/pull` rather than `/vault/stream/...`. **All five gates green**: `./scripts/build-all.sh` reports `OK (8 subdirectories)` with **nakli-local-bridge (binary built)** added; `./scripts/test-conformance.sh` 32/32; `./scripts/cli-gate.sh` 32/32; `./scripts/worker-gate.sh` 32/32; `./scripts/js-gate.sh` 3/3 across Chromium + Firefox + WebKit; **`./scripts/local-network-gate.sh` reports sync round-trip OK between two Hubs.** **Next: M8** — Saanjha (shared-list consumer) per [`shared-list-spec-001-v1.0.md`](docs/specs/shared-list-spec-001-v1.0.md). Estimate per handoff: 4–5 sessions.

## 2026-05-17 — M6 complete: Cloudflare Worker transport

M6 lands [`nakli-cf-worker/`](nakli-cf-worker/) — the zero-ops alternative transport per [`cf-worker-spec-001-v1.1.md`](docs/specs/cf-worker-spec-001-v1.1.md). A single TypeScript Worker (`src/worker.ts`, ~700 LOC) backed by Cloudflare R2 (blobs) and KV (state) implements the full protocol surface in parallel with the Hub. The Worker uses raw `fetch`-style dispatch (Hono is in deps but unused — the spec calls out Hono as the framework choice, but the route count is small enough that a single switch is more transparent and bundles smaller). Supporting modules: [`src/envelope.ts`](nakli-cf-worker/src/envelope.ts) (envelope helpers + CORS), [`src/macaroon.ts`](nakli-cf-worker/src/macaroon.ts) (mint/parse/verify via the `macaroon` npm package — wire-compatible with the Hub's Go-signed macaroons), [`src/caveats.ts`](nakli-cf-worker/src/caveats.ts) (caveat catalogue ported from `nakli-hub/internal/server/caveats.go`), [`src/storage.ts`](nakli-cf-worker/src/storage.ts) (KV + R2 layout per spec §Storage), [`src/hash.ts`](nakli-cf-worker/src/hash.ts) (SHA-256 via WebCrypto). **Caveat enforcement** is at parity with the Hub: all thirteen caveat types are wired, including `agent-id`/`device-id`/`principal-type` cross-checks against `X-Fabric-Agent-Id`/`X-Fabric-Device-Id`/`X-Fabric-Principal-Type` headers, an in-memory token-bucket `rate` enforcer (per-isolate — documented limitation for v1.0), the `requires-human-approval` 202 + `pending_id` short-circuit with `/bridge/pending/{id}` retrieval, and `discharge-from` via `POST /grant/discharge` + `X-Fabric-Discharge` header parsing. **Storage** uses R2 for blob content (object key `blobs/<ns>/<sid>/<event_id>.bin` with R2 customMetadata recording kind/sequence/principal) and KV for everything else (principals, streams, events, idempotency, grant revocation cache, pending Bridge ops, pairing tokens). **Conformance shape adjustments**: `vault.subscribe` is 501 (SSE lands at M6.x via Durable Objects); `sync.*` is 501 (multi-anchor sync is M7); Bridge dispatch is limited to the inert `conformance-test` noop adapter (real adapter execution belongs on the Hub — the Worker can `fetch()` outward but the v1.0 reference deliberately keeps adapters off the edge to avoid bloating the bundle). A `POST /fabric/v1/_conformance/setup` endpoint, gated behind `CONFORMANCE_MODE=true`, seeds the retired-agent principal test 30 expects. **Gate** (handoff §M6): [`./scripts/worker-gate.sh`](scripts/worker-gate.sh) builds nakli-hub (for the conformance runner) + the Worker bundle, generates a fresh Hub identity, writes `.dev.vars`, runs `wrangler dev --local` (Miniflare-simulated R2 + KV — same path Cloudflare documents for CI), waits for `/health`, then runs `nakli-hub conformance --target http://127.0.0.1:18934`. **32/32 passing in ~140ms.** Deployed-to-Cloudflare conformance is a one-step `wrangler deploy` + the same conformance command (documented in the Worker README); we don't run that path in CI because it requires a real Cloudflare account, but the build is deploy-ready. `./scripts/build-all.sh` now reports **nakli-cf-worker (typecheck clean)** alongside the other subdirectories. **Next: M7** — Local Network transport per [`local-network-spec-001-v1.1.md`](docs/specs/local-network-spec-001-v1.1.md): mDNS-based peer discovery, multi-anchor sync (which makes the Hub's currently-501 `sync.*` endpoints live), browser-tool integration via the local-bridge daemon. Estimate per handoff: 2–3 sessions.

## 2026-05-17 — M5.5 complete: Bridge adapter framework + 8 starter adapters

M5.5 lands the Bridge adapter framework per [`bridge-adapters-spec-001-v1.1.md`](docs/specs/bridge-adapters-spec-001-v1.1.md). The interface lives in [`fabric-sdk-go/bridge/`](fabric-sdk-go/bridge/) (`Adapter`, `OperationSpec`, `ParamSpec`, `CallRequest`, `CallResponse`, `CallMetrics`, `Registry`); each adapter is a stateless module under `bridge/adapters/<name>/`. Eight starter adapters ship: **`courtlistener`** (US court records — `search`, `get-opinion`, `get-docket`, optional API key), **`archive-org`** (Wayback + advanced search + item metadata, anonymous), **`nasa-images`** (Mars rover photos + image library, free API key), **`webhook-post`** (generic JSON POST; spec calls out `only-domain` or `requires-human-approval` as load-bearing for safe agent use), **`email-resend`** (Resend.com send, bearer auth, native `Idempotency-Key` header), **`anthropic-claude`** (Messages API, `x-api-key` + `anthropic-version` headers), **`openai-compatible`** (OpenAI/OpenRouter/Together/vLLM/LM Studio with per-call `base_url` override), and **`cloudflare-r2`** (S3-compatible PUT/GET/LIST/DELETE with an inline AWS SigV4 implementation in [`sigv4.go`](fabric-sdk-go/bridge/adapters/cloudflarer2/sigv4.go) — chose ~150 LOC of HMAC chain over a ~3 MB aws-sdk-go-v2 dependency for what is, in this adapter, a handful of HTTP calls). All eight adapters use plain `net/http`; the spec's recommendation to "use the canonical SDK if it adds genuine value" pointed at lightweight adapters not deserving the SDK weight. Each adapter dispatches on operation FIRST before resolving credentials — the conformance runner's unknown-operation check would otherwise misfire. A generic [`bridge.ConformanceTest`](fabric-sdk-go/bridge/conformance.go) runner validates the eight spec-mandated properties (interface basics, happy-path per operation, unknown-operation rejection, cancellation, no credential persistence) using an `httptest`-backed fixture per adapter — every adapter ships a `_test.go` that wires this up. Inert [`bridge.NoopAdapter`](fabric-sdk-go/bridge/noop.go) (registered name: `conformance-test`) lets the conformance suite drive `/bridge/call` for caveat tests 12/13 without depending on real upstream services.
