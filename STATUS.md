## 2026-05-17 — M4 complete: reference CLI

M4 lands the operator/developer CLI per [`cli-spec-001-v1.1.md`](docs/specs/cli-spec-001-v1.1.md). The binary is `nakli-cli`, built with cobra (subcommands + auto-generated `completion` for bash/zsh/fish/powershell) and viper (TOML config + env-var binding). Module path: `github.com/NakliTechie/private-mesh/nakli-cli`, added to `go.work`. The CLI is a thin shell over [`fabric-sdk-go`](fabric-sdk-go/) and the M3 conformance suite; no protocol speaking lives in the CLI itself. Package layout: [`cmd/nakli-cli/`](nakli-cli/cmd/nakli-cli/) (entrypoint, sets `BinaryVersion`); `internal/cmd/` (cobra command tree); `internal/config/` (TOML config via viper, with `~/.config/nakli-cli/config.toml` defaults); `internal/fifio/` (FIF create / load / unlock + passphrase prompts via `golang.org/x/term`); `internal/httpc/` (Hub HTTP client + envelope parsing + idempotency-key generation); `internal/output/` (the human-vs-JSON output split mandated by spec §Conventions). **Gate-critical commands wired**: `init` (interactive + `--non-interactive` for scripted runs; writes FIF + TOML config + registers a Hub transport), `identity init` / `identity show`, `transport add` / `list` / `remove` / `ping`, `grant mint` / `inspect` (mint reads `hub-identity.json` via the transport's `hub_data_dir` for the macaroon root key — the M4 bootstrap path), `vault append` (reads stdin or `--payload-file`, POSTs to Hub) / `vault read` (decodes payload_ciphertext back to bytes for human mode), `conformance` (wraps `fabric-sdk-go/conformance.RunAll`; pre-seeds the retired-principal row test 30 expects via a 4-line SQLite UPSERT), `status` (CLI config + transport reachability), `version` / `generate-ulid` / `generate-hub-identity`. **Deferred commands** ship as stubs that print a clear pointer at the milestone where they land — `identity pair/agents`, `grant verify/revoke/list`, `vault subscribe`, `history/bridge/llm/queue/backup/restore`. Every gate-critical command honors `--json` (single-envelope shape `{ok, data}` or `{ok:false, error:{code,message}}`), `--quiet`, `--verbose`, `--passphrase-stdin`, `--transport`, and the spec's exit-code semantics (cobra surfaces non-zero on `RunE` errors). **Gate** (handoff §M4): [`./scripts/cli-gate.sh`](scripts/cli-gate.sh) builds both binaries, starts the Hub on a free port with an unreachable peer, runs `nakli-cli init --non-interactive` → `grant mint` → `vault append` → `vault read` (asserts the event round-trips with the payload decoded) → `conformance` (asserts `32/32 passing`). All steps pass end-to-end. `./scripts/build-all.sh` reports `build-all: OK (8 subdirectories)` with **nakli-cli: 6 tests passing + binary built** alongside the existing nakli-hub: 24 / fabric-sdk-go: 29. **Next: M5** — JS SDK per [`fabric-sdk-js-spec-001-v1.1.md`](docs/specs/fabric-sdk-js-spec-001-v1.1.md): all primitives + IndexedDB queue + Web Locks leader election + SSE subscribe + headless-browser conformance. Estimate per handoff: 2–3 sessions.

## 2026-05-16 — M3 complete: 32/32 conformance

M3 lands the spec's 32-test conformance suite and turns every Phase-1 caveat into an enforced check. The suite lives in `fabric-sdk-go/conformance/` as a transport-agnostic Go package: `RunAll(cfg)` mints synthetic Grants with the target Hub's macaroon root key (read from `hub-identity.json`), drives the 32 tests over HTTP, and returns structured `Results` the CLI can pretty-print or emit as JSON. Tests are split into the six spec groups (wire-format 1–5, Grant 6–16, idempotency 17–20, Vault/History 21–24, failure-model 25–27, adversarial 28–32) — each numbered exactly as in [`fabric-spec-001-v1.0.md`](docs/specs/fabric-spec-001-v1.0.md). On the Hub side, the parsed-but-not-enforced caveats from M2 are now wired: **rate** uses a mutex-guarded in-memory token bucket per `grant_id` (returns 429 `rate_limited` on exhaustion); **max-amount** and **only-domain** are evaluated on `/bridge/call` with the call's `amount/currency/domain`; **requires-human-approval** intercepts the bridge handler before its M5.5 stub, inserts a `pending_bridge` row, and short-circuits with HTTP 202 + `human_approval_required` + a `pending_id` accessible at the new `GET /bridge/pending/{id}` endpoint; **discharge-from** is honored end-to-end via a new `POST /grant/discharge` endpoint that mints fresh discharge macaroons (Hub is its own verifier), an `X-Fabric-Discharge` header the auth middleware parses, and a refusal to mint discharges for revoked Grants. New adversarial-test prerequisites land too: `/grant/mint` enforces a caveat-superset rule when a `parent_grant_macaroon` is supplied (test 32) and a scope-narrowing rule (test 15); `storage.RetirePrincipal` plus an auth-middleware check enforce agent retirement via the `X-Fabric-Agent-Id` header (test 30); Grant binding to a specific agent uses the existing `agent-id == <id>` caveat against the same header (test 29); and `MarkGrantRevoked` now upserts a stub row so out-of-band-minted Grants are still tracked once revoked (test 16). `/health` gained a peer-reachability probe (TCP/HTTP with a 750ms timeout) whose results drive the `degraded` flag and `degraded_reasons` list — required by test 26. `nakli-hub serve` accepts a repeatable `--peer-url` flag so `scripts/test-conformance.sh` can boot a Hub with an unreachable peer for the gate run. `nakli-hub conformance --target ... --data-dir ...` replaces the M2 stub: it loads `hub-identity.json`, opens the Hub's SQLite to pre-populate the retired-principal row test 30 expects, then invokes `conformance.RunAll` and exits non-zero on any failure. **Gate** (handoff §M3): `./scripts/test-conformance.sh` reports `32/32 passing` and exits 0; the in-process `TestConformanceSuite` in [`nakli-hub/internal/server/conformance_test.go`](nakli-hub/internal/server/conformance_test.go) reaches the same number; `./scripts/build-all.sh` reports **nakli-hub: 24 tests passing + binary built** and **fabric-sdk-go: 29 tests passing**; the two-terminal manual gate (`nakli-hub serve --peer-url ...` in one shell + `nakli-hub conformance --target ...` in another) lands 32/32 in ~120ms. **Next: M4** — Reference CLI per `cli-spec-001-v1.1.md` (commands + JSON output + config/env + shell completions). Estimate per handoff: 2–3 sessions.
